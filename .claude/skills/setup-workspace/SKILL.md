# 워크스페이스 초기 세팅

CLAUDE.md의 빈칸(`___`)을 채우고, 사용자 역할에 맞는 폴더 구조를 만들어주는 대화형 세팅 도구.

---

## 실행 조건

다음 중 하나에 해당하면 실행:
- 사용자가 `/setup` 입력
- CLAUDE.md에 `___` 빈칸이 있는 상태에서 "세팅", "설정", "시작" 등 언급
- 처음 대화를 시작하면서 "뭘 해야 해?" 같은 질문

---

## 실행 흐름

### Step 1: 환영

```
안녕하세요! 워크스페이스 초기 세팅을 도와드릴게요.
질문 3개만 답해주시면 바로 시작할 수 있어요. 😊
```

---

### Step 2: 질문 (하나씩, 절대 한 번에 여러 개 묻지 말 것)

**Q1:**
```
이름이 뭐예요?
```

**Q2:**
```
어떤 일을 하고 계세요?
(예: 마케터, 카페 운영, 디자이너, 프리랜서, 학생, 사무직...)
```

**Q3:**
```
반복하는 업무 하나만 말해주세요.
(예: 주간 보고서 쓰기, 회의록 정리, SNS 포스팅, 거래처 연락, 재고 확인...)
```

**중요**: 
- 반드시 **하나씩** 질문. 한꺼번에 묻지 않는다.
- 답변이 짧아도 OK. 추가 질문하지 않는다.
- 3개 다 답하면 바로 다음 단계로.

---

### Step 3: CLAUDE.md 생성

1. 현재 CLAUDE.md 파일을 읽는다 (`./CLAUDE.md`)
2. 빈칸(`___`)을 답변으로 채운다
3. 응답 규칙은 기본값으로 세팅 (사용자에게 묻지 않고)
4. 반복 업무 칸에 Q3 답변을 넣는다

**채울 내용:**

```markdown
# CLAUDE.md — 내 업무 워크스페이스

## 사용자 프로필

- 이름: {Q1 답변}
- 역할: {Q2 답변}
- 주요 업무: {Q2 답변 기반으로 한 줄 설명}

## 이 워크스페이스의 목적

- {Q2 기반으로 자연스럽게 한 줄 작성. 예: "마케팅 업무 정리 및 콘텐츠 관리"}

## 반복 업무

- {Q3 답변}

## 응답 규칙

- 한국어로 대화
- 간결하게 답변 (길게 설명하지 않기)
- 파일 수정/삭제 전에 반드시 확인
- 톤: 편하게 (반말 OK)

## 폴더 구조

```
00-inbox/       - 빠른 캡처 (임시 보관)
10-work/        - 진행 중인 업무
20-resources/   - 참고 자료
30-personal/    - 개인 메모
90-archive/     - 완료된 업무
```

**네이밍 규칙**: `[숫자]-[설명적-이름]`

## 사용 가능한 커맨드

| 커맨드 | 용도 |
|--------|------|
| /daily-note | 오늘 메모 생성 |
| /thinking-partner | 생각 정리 도우미 |
| /status | 현재 상태 확인 |
| /inbox | 인박스 확인 |
| /meeting | 회의록 작성 |
| /weekly-review | 주간 리뷰 |

---

*이 파일은 쓰면서 키워가세요!*
*"이거 CLAUDE.md에 추가해줘"라고 말하면 됩니다.*
```

---

### Step 4: 폴더 구조 제안

Q2 답변(역할)을 기반으로 `10-work/` 아래에 하위 폴더를 **2-3개만** 제안한다.

**제안 예시:**

| 역할 | 제안 폴더 |
|------|----------|
| 마케터 | `10-work/campaigns/`, `10-work/reports/`, `10-work/content/` |
| 카페 운영 | `10-work/menu/`, `10-work/suppliers/`, `10-work/daily-ops/` |
| 디자이너 | `10-work/projects/`, `10-work/assets/`, `10-work/clients/` |
| 프리랜서 | `10-work/clients/`, `10-work/invoices/`, `10-work/portfolio/` |
| 사무직 | `10-work/reports/`, `10-work/meetings/`, `10-work/tasks/` |
| 학생 | `10-work/courses/`, `10-work/assignments/`, `10-work/notes/` |
| 연구원 | `10-work/experiments/`, `10-work/papers/`, `10-work/data/` |

위 표에 정확히 맞지 않으면 **역할에서 유추**해서 2-3개 제안.

```
{이름}님은 {역할}이시니까, 이런 폴더가 있으면 편할 것 같아요:

- 10-work/{폴더1}/ — {설명}
- 10-work/{폴더2}/ — {설명}
- 10-work/{폴더3}/ — {설명}

이대로 만들까요?
```

사용자가 OK하면 폴더 생성. 수정 요청하면 반영.

---

### Step 5: 완료 안내

```
세팅 완료! 🎉

만들어진 것:
- ✅ CLAUDE.md (당신의 프로필 + 작업 규칙)
- ✅ 10-work/ 폴더 구조

이제 이렇게 말해보세요:

> "{Q3 답변을 기반으로 한 첫 시도 프롬프트}"

예를 들어 Q3가 "주간 보고서"였다면:
> "주간 보고서 양식을 만들어줘"

💡 팁: CLAUDE.md는 한 번에 완성하는 게 아니에요.
쓰다가 "아 이건 매번 말하기 귀찮다" 싶으면
"이거 CLAUDE.md에 추가해줘"라고 하면 됩니다.
```

---

## 주의사항

- **절대 한 번에 여러 질문 묻지 말 것** — 하나씩만
- **추가 질문으로 파고들지 말 것** — 3개면 충분. 더 물으면 부담
- **기존 CLAUDE.md에 내용이 있으면** 빈칸만 채우고 나머지는 건드리지 않기
- **폴더 생성 전 반드시 확인** — "이대로 만들까요?" 물어보기
- **톤은 친근하게** — 워크샵 초보자 대상. 기술 용어 최소화
