오늘 작업한 내용을 git에서 자동 수집해서 정리해주세요.

**오늘 날짜**: $CURRENT_DATE

**수행할 작업:**

1. 다음 git 명령들을 실행해서 정보 수집:
   - `git status` — 수정/추가/삭제된 파일, untracked 파일 목록
   - `git diff --stat` — 변경 규모 파악 (unstaged)
   - `git diff --cached --stat` — staged 변경 규모
   - `git log --since="today" --oneline` — 오늘 커밋 목록 (없어도 OK)

2. 각 파일의 목적·변경 맥락 파악:
   - **신규 파일**: Read로 내용을 읽어서, 그 파일이 **무엇을 위한 파일인지** 한 줄로 파악
   - **수정된 파일**: `git diff <파일경로>`로 변경 내용을 읽어서, **오늘 뭘 바꿨는지** 한 줄로 파악
   - **삭제된 파일**: 읽을 수 없으므로 상태만 표시
   - `.DS_Store` 등 시스템 파일은 스킵 (목록에서 제외)

3. 수집 결과 분석:
   - 변경된 파일들을 (신규), (수정), (삭제) 등으로 분류
   - 각 파일에 **— 한 줄 설명**을 붙여서 목적을 명시
   - 커밋이 있으면 커밋 메시지 목록 정리
   - 위 내용을 2~3문장으로 자연스럽게 요약

4. `30-personal/daily/$CURRENT_DATE.md` 파일 확인:
   - **이미 있으면**: `## 작업 로그` 섹션을 추가 (기존 내용 보존)
   - **없으면**: 아래 템플릿으로 새 파일 생성

5. 아무 변경도 없으면 (git status가 clean하고 오늘 커밋도 없으면):
   - "오늘은 이 워크스페이스에서 작업한 기록이 없어." 라고 안내하고 종료

**작업 로그 형식:**

```markdown
## 작업 로그
> 자동 생성: $CURRENT_DATE

### 변경된 파일
- `파일경로` (신규) — 이 파일의 목적/역할 한 줄 설명
- `파일경로` (수정) — 이 파일의 목적/역할 한 줄 설명
- `파일경로` (삭제)

### 커밋
- 커밋 메시지 1
- 커밋 메시지 2

### 요약
(수집한 내용을 2~3문장으로 자연스럽게 요약)
```

**daily note가 없을 때 생성할 템플릿:**

```markdown
# $CURRENT_DATE

## 작업 로그
> 자동 생성: $CURRENT_DATE

### 변경된 파일
(위에서 수집한 내용 — 각 파일에 한 줄 설명 포함)

### 커밋
(있으면 표시, 없으면 섹션 생략)

### 요약
(요약)

## 메모

```

**주의사항:**
- 기존 daily note 내용은 절대 삭제하지 않기
- `## 작업 로그` 섹션이 이미 있으면, 내용을 최신으로 교체
- 커밋이 없는 날에도 동작 — unstaged 변경사항만으로도 기록
- Bash 도구는 git 명령만 실행할 것

**출력:**
- 수집한 정보를 먼저 보여주고
- daily note에 저장했으면: "작업 로그 저장 완료: 30-personal/daily/$CURRENT_DATE.md"
